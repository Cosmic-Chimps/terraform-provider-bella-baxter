# Dokploy on AWS EC2 — Terraform Setup

Provisions an EC2 instance on **us-east-1**, installs **Dokploy** (Docker-based PaaS), and automatically registers your GitHub Docker Compose repos for deployment.

## What gets created

| Resource | Details |
|---|---|
| VPC + Subnet + IGW | Isolated network, public subnet |
| Security Group | Ports 22 (SSH), 80, 443, 3000 (Dokploy UI) |
| EC2 t3.medium | Ubuntu 22.04, Docker, Docker Compose, Dokploy |
| Elastic IP | Static public IP |
| SSH Key Pair | Generated by Terraform, saved as `dokploy.pem` |

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) ≥ 1.3
- AWS credentials configured (`aws configure` or env vars)
- GitHub repos with a `docker-compose.yml` at the specified path

## Quick start

```bash
# 1. Clone / copy this directory
cd dokploy-terraform

# 2. Create your variables file
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars — fill in email, password, repo URLs

# 3. Init & apply
terraform init
terraform apply

# 4. Open the Dokploy UI (printed in outputs)
#    http://<public-ip>:3000
```

## terraform.tfvars reference

```hcl
dokploy_admin_email    = "you@example.com"
dokploy_admin_password = "StrongPassword123!"
github_token           = "ghp_..."   # leave "" for public repos

docker_compose_apps = [
  {
    name         = "my-app"
    repo_url     = "https://github.com/your-org/your-app"
    branch       = "main"
    compose_file = "docker-compose.yml"
    description  = "My app"
  }
]
```

## Security notes

- **`allowed_ssh_cidr`** — change from `0.0.0.0/0` to your IP (`["x.x.x.x/32"]`) before applying.
- **`dokploy.pem`** — the private key is written locally. Keep it safe and never commit it.
- **Secrets** — use `terraform.tfvars` (git-ignored) or environment variables (`TF_VAR_dokploy_admin_password`).

## Useful commands after deploy

```bash
# SSH into the server
ssh -i dokploy.pem ubuntu@<public-ip>

# View Dokploy container logs
ssh -i dokploy.pem ubuntu@<public-ip> "docker logs dokploy --tail 100 -f"

# View bootstrap log
ssh -i dokploy.pem ubuntu@<public-ip> "cat /var/log/user-data.log"

# View app registration log
ssh -i dokploy.pem ubuntu@<public-ip> "cat /home/ubuntu/configure_dokploy.log"
```

## Tear down

```bash
terraform destroy
```

This removes all resources including the Elastic IP and EC2 instance. Data on the instance is lost.

## Adding more apps later

Add a new block to `docker_compose_apps` in `terraform.tfvars` and run `terraform apply` again. The `null_resource` will re-run the configuration script.
